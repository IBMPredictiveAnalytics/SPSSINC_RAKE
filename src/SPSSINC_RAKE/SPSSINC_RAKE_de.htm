Raking für Gewichtungen und Kontrollsummen durchführen

Diese Prozedur berechnet Fallgewichtungen zum Abgleich von Kontrollsummen für Kategorien von bis zu fünf Variablen. Wenn das Dataset bereits gewichtet ist, umfasst die neue Gewichtungsvariable die vorhandene Gewichtung.

Zu erstellende Gewichtungsvariable. Geben Sie die Gewichtungsvariable an, die erstellt werden soll.

• Der Name der Gewichtungsvariablen darf nicht bereits verwendet werden.

Kontrollsummenvariable. Wählen Sie mindestens eine Kontrollvariable aus.

• Es können nur numerische Kategorievariablen verwendet werden. Mit AUTORECODE können Sie eine numerische Version einer Zeichenfolgevariablen erstellen.

• Bis zu fünf Kontrollvariablen können in diesem Dialogfeld angegeben werden (über die Syntax können mehr angegeben werden). Je mehr Variablen verwendet werden, umso schwieriger wird jedoch der Abgleich der Kontrollsummen und umso wahrscheinlicher wird das Auftreten extremer Gewichtungen.

Kategorien und Kontrollsummen oder Bruchteile. Für jede der angegebenen Kontrollvariablen geben Sie eine Liste von Kategorienwerten und Gesamtsummen oder Bruchteilen getrennt durch ein Leerzeichen ein. Beispiel:
1 .5 2 .3 3 .2
Kategorie 1 sollte mit der Hälfte der Stichprobe gewichtet werden; Kategorie 2 mit 30% und Kategorie 3 mit 20%.
1 500 2 300 3 200
Dieser Ausdruck ist die Entsprechung mit Anzahlen anstelle von Bruchteilen.

Die Kontrollsumme kann auch ein numerischer Ausdruck sein. Wird ein Ausdruck verwendet, setzen Sie den Ausdruck in Anführungszeichen. Beispiel:

1 "1594 / (1594 + 603)" 2 ".87 *.28"

Dies kann sinnvoll sein, wenn Sie verschachtelte Kategorien haben und Dimensionen ausblenden müssen.

Werden die Werte über ein Dataset angegeben, kann die Kontrollsummenvariable eine Zeichenfolge sein, die den Ausdruck enthält.

• Bei Verwendung von Syntax können die Gewichtungen aus einem Dataset kommen.

• Verwenden Sie als Dezimaltrennzeichen immer einen Punkt.

Diese Prozedur generiert Gewichtungen, sodass die Randsummen für jede Kontrollvariablenkategorie so genau wie möglich den angegebenen Gesamtsummen entsprechen.

Grundgesamtheitssumme. Optional können Sie eine Grundgesamtheitssumme angeben, auf die sich die gewichteten Fälle summieren.

• Fälle können nur aus vorhandenen Werten erstellt werden: wenn Sie Steuerwerte angeben, die im Dataset nicht auftreten, ist die gewichtete Anzahl von Fällen geringer als angefordert.

Optionen

Der Dialog Optionen gibt Kriterien für den GENLOG-Schätzungsprozess an, der von dieser Prozedur verwendet wird. Außerdem wird festgelegt, ob die GENLOG-Ausgabe angezeigt oder ausgeblendet werden soll, und das generierte Hilfsdataset wird angegeben.

Schätzkriterien. Die Schätzung wird iterativ von GENLOG durchgeführt. Die Newton-Raphson-Methode wird verwendet, um Maximum-Likelihood-Schätzungen zu erhalten. Sie können neue Werte für die maximale Anzahl von Iterationen, das Konvergenzkriterium und einen Deltawert (eine Konstante, die für anfängliche Approximationen zu allen Zellen hinzugefügt wird) angeben.

• Wenn die berechneten Gewichtungen keine Ergebnisse nahe den angegebenen Anteile ergeben, versuchen Sie, DELTA=0 zu definieren.

GENLOG-Ausgabe und Hilfsdataset anzeigen. Wählen Sie dieses Feld aus, um die GENLOG-Ausgabe und das bei der Berechnung der Gewichtungen verwendete Hilfsdataset anzuzeigen.

Diagramme

Histogramme der Gewichtungen anzeigen. Wählen Sie dieses Feld aus, um ein nicht gewichtetes Histogramm der berechneten Gewichtungen anzuzeigen, die für alle Fälle summiert wurden. Dies kann beim Prüfen auf extreme Gewichtungswerte hilfreich sein.

Heat-Map. Eine Heat-Map kann für bis zu vier Dimensionenen angezeigt werden. Die den Dimensionen zugewiesenen Variablen werden in der Reihenfolge verwendet, in der Sie im Hauptdialogfeld aufgelistet werden. Sie werden den Y- und X-Dimensionen und anschließend entsprechend der Anzahl der angegebenen Dimensionen der Spalten- und Zeileneinteilung zugewiesen. Die Anzahl der Dimensionen entspricht maximal der Gesamtanzahl der Kontrollvariablen. Wenn es nur eine Kontrollvariable gibt, ist kein Plot verfügbar.

Die dunklen Bereiche in der Heat-Map geben an, wo sich die großen Gewichtungen befinden. Dieser Plot kann also ermitteln, wo die Gewichtungen die Fälle am meisten verstärken.

• Wenn es mehr Kontrollvariablen gibt, als in der Heat-Map angezeigt werden, sind die Heat-Map-Statistikdaten die Gewichtungsdaten für jede Zelle, die in der Map als Durchschnitt des relevanten Bereichs definiert ist.

• In der Syntax haben Sie mehr Kontrolle über die Plotdimensionen.

Weitere Funktionen

Die Syntax kann bis zu zehn Dimensionen enthalten, wenn Sie ausreichend Hauptspeicher und Geduld haben. Zeitaufwand und Speicherbedarf steigen rapide mit der Anzahl der Dimensionen.

In der Syntax können die Kontrollsummen aus einem Dataset übernommen werden.

In der Syntax können Sie die Tabelle der Gewichtungen unterdrücken, was bei einer großen Anzahl unterschiedlicher Gewichtungen empfohlen wird.

In der Syntax können Sie die Heat-Map-Dimensionsvariablen explizit auswählen und steuern, welchen Achsen sie zugewiesen werden.

Führen Sie den folgenden Befehl aus:
SPSSINC RAKE /HELP.
Drücken Sie alternativ die Taste F1 für eine Befehlsinstanz (Statistics Version 23 oder später), um die gesamte Syntaxbeschreibung anzuzeigen.

Anforderungen

Für diesen Befehl sind Python Essentials und die Option Advanced Statistics erforderlich.
