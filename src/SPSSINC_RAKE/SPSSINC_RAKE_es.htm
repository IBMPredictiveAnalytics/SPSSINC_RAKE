Ponderaciones Rake para totales de control

Este procedimiento calcula las ponderaciones de caso para que coincidan con los totales de control para las categorías de una a cinco variables. Si el conjunto de datos ya está ponderado, la nueva variable de ponderación incorpora la ponderación existente.

Variable de ponderación para crear. Especifique la variable de ponderación que se va a crear.

• El nombre de la variable de ponderación aún no debe estar en uso.

Variable total de control. Seleccione, como mínimo, una variable de control.

• Sólo se pueden utilizar variables de categoría numéricas. Puede utilizar Autorecode para crear una versión numérica de una variable de cadena.

• Se pueden especificar hasta cinco variables de control utilizando el diálogo (se pueden especificar más en la sintaxis), pero cuantas más se utilicen, más difícil será que coincidan con los totales de control y mayor será la probabilidad de que se produzcan ponderaciones extremas.

Categorías y totales de control o fracciones. Para cada una de las variables de control que especifique, entre una lista de valores de categoría y totales o fracciones separados por un espacio en blanco. Por ejemplo,
1 .5 2 .3 3 .2
La categoría 1 se debe ponderar en la mitad de la muestra; la categoría 2 en un 30%, y la categoría 3 en 20%.
1 500 2 300 3 200
sería el equivalente expresado como recuentos, en lugar de como fracciones.

El total de control también puede ser una expresión numérica. Si se utiliza una expresión, especifique la expresión entre comillas. Por ejemplo,

1 "1594 / (1594 + 603)" 2 ".87 *.28"

Esto puede ser útil si tiene categorías anidadas y necesita contraer dimensiones

Si los valores se han especificado utilizando un conjunto de datos, la variable de total de control puede ser una cadena que contiene la expresión.

• Mediante el uso de la sintaxis, puede tener ponderaciones procedentes de un conjunto de datos.

• Utilice siempre un punto como separador decimal.

El procedimiento generará ponderaciones de forma que los totales marginales para cada categoría de variable de control coincida con los totales especificados lo más cerca posible.

Total de población. De forma opcional, puede especificar un total de población que será la suma de los casos ponderados.

• Los casos no se puede fabricar de la nada: si especifica valores de control que no se producen en el conjunto de datos, el número ponderado de casos será menor que el solicitado.

Opciones

El diálogo Opciones especifica los criterios para el proceso de estimación de Genlog utilizado por este procedimiento y si se va a ocultar o mostrar el resultado del Genlog y el conjunto de datos auxiliar que se va a generar.

Criterios de estimación. Genlog realiza la estimación de forma iterativa. El método Newton-Raphson se utiliza para obtener las estimaciones de parámetro de máxima verosimilitud. Puede entrar nuevos valores para el número máximo de iteraciones, el criterio de convergencia y delta (una constante añadida a todas las casillas para aproximaciones iniciales).

•  Si las ponderaciones calculadas no producen resultados cercanos a las proporciones especificadas, intente establecer DELTA=0.

Mostrar el resultado de Genlog y el conjunto de datos auxiliar. Marque este recuadro para ver el resultado de Genlog y el conjunto de datos auxiliar utilizados en el cálculo de las ponderaciones.

Gráficos

Mostrar histograma de ponderaciones. Active este recuadro para mostrar un histograma no ponderado de las ponderaciones calculadas acumuladas en todos los casos. Esto puede resultar de utilidad para comprobar los valores de ponderación extremos.

Mapa de calor. Se puede mostrar un mapa de calor para cuatro dimensiones como máximo. Las variables asignadas a las dimensiones se toman en el orden en que aparecen enumeradas en el recuadro de diálogo principal y se asignan a las dimensiones "y" y "x" y, a continuación, a los paneles de columna y fila según el número de dimensiones especificado. El número de dimensiones es como máximo el número total de variables de control. No hay ningún gráfico disponible si hay solo una variable de control.

Las áreas oscuras en el mapa de calor indican dónde se encuentran las ponderaciones grandes, de modo que este gráfico puede determinar dónde las ponderaciones amplifican al máximo los casos.

• Si hay más variables de control que las que se muestran en el mapa de calor, las estadísticas son las ponderaciones en cada casilla definida en el mapa promediadas sobre las relevantes.

• En la sintaxis tiene más control sobre las dimensiones del gráfico.

Características adicionales

En la sintaxis puede tener hasta diez dimensiones, si tiene suficiente memoria y paciencia. Los requisitos de tiempo y, también, de memoria aumentan rápidamente con el número de dimensiones.

En la sintaxis, los totales de control se pueden tomar de un conjunto de datos.

En la sintaxis, puede suprimir la tabla de ponderaciones, que se recomienda si va a haber un gran número de ponderaciones distintas.

En sintaxis, puede elegir explícitamente las variables de dimensión del mapa de calor y controlar los ejes a los que están asignadas.

Ejecute el comando
SPSSINC RAKE /HELP.
o pulse F1 en una instancia de mandato (Statistics V23 o posterior) para mostrar la descripción de sintaxis completa.

Requisitos

Este comando requiere Python Essentials y la opción Estadísticas avanzadas.
