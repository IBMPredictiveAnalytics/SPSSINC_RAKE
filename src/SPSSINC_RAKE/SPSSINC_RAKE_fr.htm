Pondérations de ratissage pour les totaux de contrôle

Cette procédure calcule des pondérations d'observation afin d'assurer la correspondance des totaux de contrôle pour des catégories d'une à cinq variables. Si le jeu de données est déjà pondéré, la nouvelle variable de pondération intègre la pondération existante.

Variable de pondération à créer Indiquez la variable de pondération à créer.

• Le nom de la variable de pondération ne doit pas déjà être utilisé.

Variable de total de contrôle Sélectionnez au moins une variable de contrôle.

• Seules les variables numériques de catégorie peuvent être utilisées. Vous pouvez utiliser la commande Autorecode pour créer une version numérique d'une variable de chaîne.

• Vous pouvez indiquer cinq variables de contrôle au maximum à l'aide de la boîte de dialogue (des variables supplémentaires peuvent être entrées dans la syntaxe), mais un nombre important de variables rend plus difficile la mise en correspondance des totaux de contrôle et augmente la probabilité d'obtenir des pondérations extrêmes.

Catégories et totaux de contrôle ou fractions Pour chacune des variables de contrôle que vous indiquez, entrez une liste de valeurs de catégorie et de totaux ou de fractions séparés par un espace. Par exemple,
1 .5 2 .3 3 .2
La catégorie 1 doit être pondérée à la moitié de l'échantillon, la catégorie 2 à 30 % et la catégorie 3 à 20 %.
1 500 2 300 3 200
serait l'équivalent exprimé sous forme de comptages plutôt que de fractions.

^Le total de contrôle peut aussi être une expression numérique. Si vous utilisez une expression, placez-la entre guillemets. Par exemple,

1 "1594 / (1594 + 603)" 2 ".87 *.28"

Cela peut être utile si vous avez des catégories imbriquées et si vous devez réduire les dimensions.

Si les valeurs sont définies à l'aide d'un jeu de données, la variable de total de contrôle peut être une chaîne contenant l'expression.

• La syntaxe permet d'utiliser des pondérations provenant d'un jeu de données.

• Utilisez toujours un point comme séparation décimale.

Cette procédure générera des pondérations afin que les totaux marginaux de chaque catégorie de variable de contrôle correspondent le plus possible aux totaux indiqués.

Total de population Vous pouvez, facultativement, indiquer un total de population auquel les observations pondérées seront égales.

• Les observations ne peuvent pas être créées à partir de rien : si vous indiquez des valeurs de contrôle qui ne sont pas présentes dans le jeu de données, le nombre d'observations pondéré sera inférieur à ce qui est demandé.

Options

La boîte de dialogue Options indique des critères pour le processus d'estimation Genlog utilisé par cette procédure et s'il faut afficher ou masquer la sortie Genlog et le jeu de données auxiliaire généré.

Critère d'estimation L'estimation est effectuée de manière itérative par Genlog. La méthode de Newton-Raphson est utilisée pour obtenir le maximum d'estimations de paramètre de probabilité. Vous pouvez entrer de nouvelles valeurs pour le nombre maximal d'itérations, les critères de convergence et le delta (constante ajoutée à toutes les cellules pour les approximations initiales).

•  Si les pondérations calculées ne génèrent pas de résultats proches des proportions indiquées, paramétrez DELTA=0.

Afficher la sortie Genlog et le jeu de données auxiliaire. Sélectionnez cette case à cocher pour afficher la sortie Genlog et le jeu de données auxiliaire utilisé pour le calcul des pondérations.

Tracés

Afficher l'histogramme des pondérations. Cochez cette case pour afficher un histogramme non pondéré des pondérations calculées accumulées pour toutes les observations. Cela peut se révéler utile pour vérifier les pondérations extrêmes.

Carte de densité. Vous pouvez afficher une carte de densité pour un maximum de quatre dimensions. Les variables affectées aux dimensions sont utilisées dans l'ordre dans lequel elles figurent dans la boîte de dialogue principale. Elles sont affectées aux dimensions y et x, puis aux colonnes et aux lignes des graphiques en panneaux selon le nombre de dimensions indiqué. Le nombre de dimensions est au plus égal au nomble de variables du total de contrôle. Aucun tracé n'est disponible s'il y a une seule variable de contrôle.

La zone sombre de la carte de densité montre les pondérations élevées. Ce tracé peut ainsi mettre en évidence les endroits où les pondérations amplifient le plus les observations.

• S'il y a plus de variable de contrôle qu'il n'en figure sur la carte de densité, les statistiques de la carte sont les moyennes des pondérations de chaque cellule définie dans la carte, calculées sur les valeurs pertinentes.

• La syntaxe permet de mieux contrôler les dimensions du tracé.

Fonctions supplémentaires

La syntaxe peut comporter jusqu'à 10 dimensions si vous disposez d'une mémoire suffisante et de suffisamment de temps. Les exigences en mémoire et en temps augmentent rapidement avec le nombre de dimensions.

Dans la syntaxe, les totaux de contrôle peuvent provenir d'un jeu de données.

La syntaxe permet de supprimer la table des pondérations, ce qui est recommandé si le nombre des pondérations distinctes doit être élevé.

La syntaxe permet de choisir de manière explicite les variables de dimension de la carte de densité et de contrôler les axes auxquels elles sont affectées.

Exécutez la commande
SPSSINC RAKE /HELP.
ou appuyez sur F1 dans une instance de commande (Statistics V23 ou version ultérieure) pour afficher la description de la syntaxe dans son ensemble.

Conditions requises

Cette commande nécessite Python Essentials et l'option Advanced Statistics.
