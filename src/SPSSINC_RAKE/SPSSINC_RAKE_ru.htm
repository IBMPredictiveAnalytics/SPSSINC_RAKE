Гребневидные веса для контрольных итогов

В этой процедуре вычисляются веса наблюдений, чтобы согласовать контрольные итоги для категорий от одной до пяти переменных. Если веса для набора данных уже существуют, новая переменная веса включает в себя существующий вес.

Переменная весов для создания. Укажите создаваемую переменную весов.

• Имя переменной не может уже использоваться.

Переменная контрольных итогов. Выберите хотя бы одну контрольную переменную.

• Можно использовать только числовые и категориальные переменные. Можно воспользоваться процедурой Автоматическая перекодировка, чтобы создать числовой вариант текстовой переменной.

• Используя это диалоговое окно, можно задать до пяти управляющих переменных (больше можно задать в синтаксисе), но чем больше переменных используется, тем труднее согласовывать контрольные итоги и тем сильнее проявляется правдоподобие экстремальных весов.

Итоги или доли категорий или контрольных переменных. Для каждой заданной вами контрольной переменной введите список значений категорий и итогов или долей, разделенных пробелами. Например,
1 .5 2 .3 3 .2
Вес категории 1 должна соответствовать половине выборки, категории 2 - 30%, а категории 3 - 20%.
1 500 2 300 3 200
- эквивалентная запись, к которой задаются количества, а не доли.

Контрольный итог может быть также численным выражением. Если используется выражение, заключите его в кавычки. Например,

1 "1594 / (1594 + 603)" 2 ".87 *.28"

Это может оказаться полезным при наличии вложенных категорий для сокращения числа измерений

Если значения задаются с использованием набора данных, переменной контрольного итога может быть строка, содержащая выражение.

• Используя синтаксис, можно брать веса из набора данных.

• В качестве десятичного разделителя всегда используйте точку.

Эта процедура будет генерировать веса, чтобы маргинальные суммы для каждой категории управляющих переменных наиболее точно совпадали с заданными суммами.

Итог совокупности. Если хотите, можно задать суммарную совокупность, в которую должны складываться все взвешенные наблюдения.

• Наблюдения нельзя произвести из ничего: если вы задаете управляющие значения, которых нет в наборе данных, взвешенное количество наблюдений будет меньшим, чем требуется.

Параметры

В диалоговом окне Опции задаются критерии для процесса оценки Genlog, используемого данной процедурой, а также опция, скрывать или показывать выходные данные Genlog и дополнительный генерируемый набор данных.

Критерии оценки. Оценка производится процессом Genlog по итерациям. Метод Ньютона-Рафсона служит для получения оценок параметров методом максимального правдоподобия. Можно ввести новые значения для максимального числа итераций, критерия сходимости и дельты (константы, добавляемой во все ячейки при начальных аппроксимациях).

•  Если вычисленные веса не дают результатов, близких к заданным пропорциям, попробуйте задать параметр DELTA=0.

Показать вывод Genlog и вспомогательный набор данных. Включите этот переключатель, чтобы увидеть выходные данные Genlog и дополнительный набор данных, используемый для вычисления весов.

Графики

Показать гистограмму весов. Включите этот переключатель, чтобы вывести невзвешенную гистограмму рассчитанных весов, накопленную по всем наблюдениям. Это может быть полезным при проверке крайних значений весов.

Тепловая карта. Тепловую карту можно показать, если измерений не больше четырех. Переменные, назначенные измерениям, рассматриваются в том порядке, в котором они перечислены в главном диалоговом окне, и назначаются измерениям y и x, а затем столбцам и строкам панели в соответствии с числом заданных измерений. Число измерений ограничено числом общих управляющих переменных. Если управляющая переменная только одна, никакие графики построить нельзя.

Темные области на тепловой карте соответствуют большим весам, так что этот график может указать, где расположены веса, которые обеспечивают максимальный вклад в наблюдения.

• Если управляющих переменных больше, чем показано на тепловой карте, статистические показатели тепловой карты представляют собой веса каждой ячейки, определенной на карте, усредненные по релевантным параметрам.

• В синтаксисе у вас есть больше возможностей управления над измерениями графика.

Дополнительные возможности

Если у вас достаточно компьютерной памяти и терпения, в синтаксисе можно задать до десяти измерений. Требуемые время и память быстро растут с увеличением числа измерений.

В синтаксисе контрольные итоги можно брать из набора данных.

В синтаксисе можно подавить таблицу весов, что рекомендуется при наличии большого числа отдельных весов.

В синтаксисе вы можете явно выбрать переменные измерений тепловой карты и управлять осями, которые им назначаются.

Введите команду
SPSSINC RAKE /HELP.
или нажмите клавишу F1 в экземпляре команды (Statistics V23 или новее), чтобы вывести описание синтаксиса целиком.

Технические требования

Для этой команды требуется Python Essentials и опция Расширенная статистика.
